# 리스크 해석서 (우선순위순)

이 문서는 현재 코드에서 확인된 리스크를 항목별로 "무슨 의미인지" 바로 이해할 수 있게 해석한 문서입니다.

## [P0] 음수 리터럴이 양수로 해석되는 문제
- 원문 요약: `-`를 숫자 토큰으로 처리하지 않고 unsupported 문자로 건너뛰어 `-1`이 `1`처럼 처리될 수 있음.
- 해석: 사용자가 음수 값을 넣거나 비교해도 시스템이 조용히 부호를 잃어버립니다.
- 영향: `INSERT`, `WHERE` 결과가 틀려지고, 눈에 띄는 에러 없이 잘못된 데이터가 저장/조회될 수 있습니다.
- 위치: `src/lexer.c:199`, `src/lexer.c:341`

## [P1] WHERE 컬럼 오타가 에러 없이 0건으로 숨겨지는 문제
- 원문 요약: SELECT projection 컬럼은 검증하지만 WHERE 컬럼은 검증하지 않음.
- 해석: `WHERE agge = 20` 같은 오타가 나면 "컬럼 없음 에러"가 아니라 그냥 결과 0건처럼 보일 수 있습니다.
- 영향: 사용자/테스터가 오타를 데이터 없음으로 오해할 수 있습니다.
- 위치: `src/executor.c:326`, `src/storage.c:95`

## [P1] `.tbl` 필드 수 불일치가 조용히 흡수되는 문제
- 원문 요약: 행의 필드 개수가 스키마와 달라도 truncate/padding으로 진행됨.
- 해석: 저장 파일이 깨져 있어도 조회 단계에서 에러 없이 일부 값이 잘리거나 빈 값으로 채워집니다.
- 영향: 데이터 손상 탐지가 늦어지고, 잘못된 결과가 정상처럼 보일 수 있습니다.
- 위치: `src/storage.c:55`, `src/storage.c:325`

## [P1] 4096바이트 초과 행 분할 읽기 문제
- 원문 요약: `fgets` 버퍼를 넘는 긴 한 줄이 여러 줄처럼 쪼개져 읽힐 수 있음.
- 해석: 원래 1개 레코드가 2개 이상 레코드처럼 처리될 수 있습니다.
- 영향: 대용량/비정상 입력에서 조회 결과 수와 내용이 왜곡됩니다.
- 위치: `src/storage.c:259`, `src/storage.c:314`

## [P1] INT 오버플로/언더플로 미검출 문제
- 원문 요약: `strtol` 사용 시 `errno`를 확인하지 않아 범위 초과 값이 통과될 수 있음.
- 해석: int 범위를 넘는 숫자가 "유효한 값"처럼 처리될 수 있습니다.
- 영향: 비교 결과, PK 중복 검사, 필터링 결과가 왜곡될 위험이 있습니다.
- 위치: `src/executor.c:19`, `src/storage.c:19`

## [P2] DATE의 달/일 유효성 미검증 문제
- 원문 요약: `YYYY-MM-DD` 형태만 확인하고 실제 달/일 범위는 확인하지 않음.
- 해석: `2026-99-99` 같은 값도 형식만 맞으면 통과할 수 있습니다.
- 영향: 날짜 기반 로직/통계에서 품질 문제가 생길 수 있습니다.
- 위치: `src/executor.c:34`

## [P2] 스키마 숫자 필드 무검증 파싱 문제
- 원문 요약: `max_length`, `nullable`, `pk`를 `atoi`로 파싱함.
- 해석: 잘못된 문자열이 들어와도 에러 없이 `0`으로 변환되어 의미가 바뀔 수 있습니다.
- 영향: 스키마 의도와 다른 제약이 조용히 적용될 수 있습니다.
- 위치: `src/schema.c:167`

## [P2] 긴 토큰의 조용한 잘림 문제
- 원문 요약: 식별자/문자열/숫자 토큰이 255바이트에서 잘려도 에러가 나지 않음.
- 해석: 긴 입력이 입력한 그대로 저장/비교되지 않을 수 있습니다.
- 영향: 값 불일치, 비교 실패, 디버깅 난이도 상승이 발생할 수 있습니다.
- 위치: `src/lexer.c:171`, `src/lexer.c:114`

## [P2] 문서-실구현 불일치 (interactive mode)
- 원문 요약: 문서에는 인터랙티브 모드가 언급되지만 실제 `main`은 `-f/-e` 없으면 help 후 종료.
- 해석: 사용자가 문서만 보고 실행하면 기대 동작과 다른 결과를 받습니다.
- 영향: 온보딩 혼선, 테스트/데모 준비 시간 증가.
- 위치: `src/main.c:219`

## [P3] 샘플 SQL 파일 따옴표 불균형 가능성
- 원문 요약: `sql/members_30.sql`에 홀수 따옴표 줄이 다수 존재해 파싱 실패 가능.
- 해석: 엔진 문제가 아니라 입력 데이터 품질 문제로 실행이 중단될 수 있습니다.
- 영향: 데모/테스트에서 재현성이 떨어지고 "엔진 버그"로 오해될 수 있습니다.
- 위치: `sql/members_30.sql:2`
